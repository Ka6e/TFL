# Лексическая структура языка Swiston --

## 1. Общие положения

-   Исходный файл текст в кодировке UTF-8.
-   Программа состоит ровно из одного файла.
-   Язык регистрозависимый.
-   Комментарии и пробельные символы игнорируются лексером.

------------------------------------------------------------------------

## 2. Классификация лексем

| Класс              | Назначение                    |
| ------------------ | ----------------------------- |
| Идентификаторы     | Имена переменных и функций    |
| Ключевые слова     | Зарезервированные слова языка |
| Литералы           | Константные значения          |
| Операторы          | Арифметические, логические, сравнения, строковые |
| Разделители        | Структурные символы           |
| Комментарии        | Игнорируются компилятором     |
| Пробельные символы | Разделяют лексемы             |

## 3. Пробельные символы

Пробельные символы не имеют семантического значения и служат для разделения лексем.

Допустимые пробельные символы:

| Символ         | Unicode |
| -------------- | ------- |
| Пробел         | U+0020  |
| Табуляция      | U+0009  |
| Перевод строки | U+000A  |


### ISO EBNF

``` ebnf
whitespace = " " | "\t" | "\n"
```

------------------------------------------------------------------------

## 4. Комментарии

Поддерживаются однострочные комментарии.

Синтаксис:

" # комментарий до конца строки "

Комментарий начинается с `#` и продолжается до перевода строки.

------------------------------------------------------------------------

## 5. Идентификаторы

### Правила
  - Начинаются с буквы латинского алфавита или `_`
  - Далее могут содержать буквы, цифры и `_`
  - Не могут совпадать с ключевыми словами

### EBNF

``` ebnf
letter      = "A"…"Z" | "a"…"z" ;
digit       = "0"…"9" ;

identifier  =
    ( letter | "_" ),
    { letter | digit | "_" } ;
```

Примеры:

x
_value
counter1

## 7. Литералы

### 7.1 Целочисленные литералы (int)
   - Десятичные
   - Знак задаётся унарным минусом на уровне выражения

``` ebnf
int_literal =
    digit,
    { digit } ;
```

Примеры:

0
42

------------------------------------------------------------------------

### 7.2 Литералы с плавающей точкой (float)
   - Обязательная дробная часть
   - Обязательная точка
   - Знак задаётся унарным минусом на уровне выражения

``` ebnf
float_literal =
    digit, { digit },
    ".",
    digit, { digit } ;
```

Примеры:

3.14
10.0

------------------------------------------------------------------------

### 7.3 Строковые литералы (string)

   - Оформление: двойные кавычки `"..."`.
   - Поддерживаются экранированные последовательности: `\n, \t и.т.п.`
   - Строковые литералы могут содержать любой Unicode code point, кроме `"` и `\` без экранирования.
   - Во время выполнения строки хранятся как .NET `string` — последовательность UTF-16 code units.

#### Краткий справочник по терминологии Unicode

| Термин               | Описание |
| -------------------- | -------- |
| Байт                 | Единица хранения данных (8 бит) |
| Code unit            | Единица кодирования в конкретной кодировке: 1 байт в UTF-8, 2 байта в UTF-16, 4 байта в UTF-32 |
| Code point           | Абстрактный номер символа в стандарте Unicode (U+0000 — U+10FFFF). Большинство символов занимают один code unit в UTF-16 (BMP, U+0000–U+FFFF); символы вне BMP (например, эмодзи) занимают два UTF-16 code units (суррогатная пара), но остаются одним code point |
| Графемный кластер    | То, что пользователь воспринимает как один «символ». Может состоять из нескольких code points: например, буква с диакритическим знаком через комбинирующий символ |

Таким образом: байт < code unit ≤ code point ≤ графемный кластер.

Встроенная функция `length(s)` возвращает количество UTF-16 code units (как `string.Length` в .NET), а `substr` индексирует по code points.

Поддерживаются escape-последовательности:

#### Escape-последовательности

| Последовательность | Значение      |
| ------------------ | ------------- |
| `\"`               | Кавычка       |
| `\\`               | Обратный слеш |
| `\n`               | Новая строка  |
| `\t`               | Табуляция     |


``` ebnf
backslash = "\" ;
quote     = '"' ;

escape_sequence =
      backslash, quote
    | backslash, backslash
    | backslash, "n"
    | backslash, "t" ;

string_character =
       любой Unicode code point кроме " и \
    | escape_sequence ;

string_literal =
    quote,
    { string_character },
    quote ;
```

Пример:

"Hello\nWorld"

------------------------------------------------------------------------

### 7.4 Булевы литералы (bool)

   - Два значения: `true` и `false`.
   - Являются ключевыми словами, не могут использоваться как идентификаторы.

``` ebnf
bool_literal = "true" | "false" ;
```

## 8. Операторы

| Категория      | Операторы               |
| -------------- | ----------------------- |
| Арифметические | `+`, `-`, `*`, `/`, `%` |
| Строковые      | `+` (конкатенация)      |
| Сравнения равенства | `==`, `!=`         |
| Сравнения отношения | `<`, `>`, `<=`, `>=` |
| Логические     | `&&`, `\|\|`, `!`        |
| Присваивание   | `=`                     |


``` ebnf
plus = "+" ;
minus = "-" ;
star  = "*" ;
slash = "/" ;
percent = "%" ;

eq_op  = "==" ;
neq_op = "!=" ;
lt_op  = "<" ;
gt_op  = ">" ;
le_op  = "<=" ;
ge_op  = ">=" ;

and_op = "&&" ;
or_op  = "||" ;
not_op = "!" ;

assign_op = "=" ;
```

## 9. Разделители

| Символ  | Назначение                             |
| ------- | -------------------------------------- |
| `{` `}` | Границы блока кода                     |
| `(` `)` | Скобки для выражений и вызовов функций |
| `,`     | Разделитель элементов                  |
| `;`     | Завершение инструкции                  |
| `:`     | Разделитель типа в объявлениях         |


``` ebnf
lparen = "(" ;
rparen = ")" ;
lbrace = "{" ;
rbrace = "}" ;
comma  = "," ;
semicolon = ";" ;
colon  = ":" ;
```

## 10. Полная EBNF-грамматика лексем

``` ebnf
letter = "A"…"Z" | "a"…"z" ;
digit  = "0"…"9" ;

identifier =
    ( letter | "_" ),
    { letter | digit | "_" } ;

int_literal =
    digit,
    { digit } ;

float_literal =
    digit, { digit },
    ".",
    digit, { digit } ;

backslash       = "\" ;
quote           = '"' ;

escape_sequence =
      backslash, quote
    | backslash, backslash
    | backslash, "n"
    | backslash, "t" ;

string_character = любой символ Unicode кроме quote и backslash
    | escape_sequence ;

string_literal =
    quote,
    { string_character },
    quote ;

bool_literal = "true" | "false" ;

comment =
    "#",
    { любой символ кроме "\n"  } ;

whitespace =
      " "
    | "\t"
    | "\n" ;

plus            = "+" ;
minus           = "-" ;
star            = "*" ;
slash           = "/" ;
percent         = "%" ;

eq_op           = "==" ;
neq_op          = "!=" ;
lt_op           = "<" ;
gt_op           = ">" ;
le_op           = "<=" ;
ge_op           = ">=" ;

and_op          = "&&" ;
or_op           = "||" ;
not_op          = "!" ;

assign_op       = "=" ;

lparen          = "(" ;
rparen          = ")" ;
lbrace          = "{" ;
rbrace          = "}" ;
comma           = "," ;
semicolon       = ";" ;
colon           = ":" ;

delimiter =
      lparen
    | rparen
    | lbrace
    | rbrace
    | comma
    | semicolon
    | colon ;
```
