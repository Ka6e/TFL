# Семантика языка Swiston

## 1. Общие свойства

Swiston — императивный, статически типизированный язык.

| Свойство                     | Описание                              |
| ---------------------------- | ------------------------------------- |
| Неявные преобразования типов | Запрещены                             |
| Передача параметров          | По значению                           |

------------------------------------------------------------------------

## 2. Базовые типы данных

### 2.1 int
   - 32-битное знаковое целое
   - Поддерживает арифметику и сравнение
   - Значение по умолчанию 0

------------------------------------------------------------------------


### 2.2 float
   - 64-битное число с плавающей точкой
   - Поддерживает арифметику и сравнение
   - Неявного преобразования int → float нет
   - Значение по умолчанию 0.0

------------------------------------------------------------------------

### 2.3 string
   - Последовательность Unicode code points (во время выполнения хранится как .NET `string` — UTF-16 code units)
   - Поддерживает конкатенацию `+` и сравнение `==`, `!=`
   - Значение по умолчанию ""

------------------------------------------------------------------------

### 2.4 bool
   - Два значения: `true`, `false`
   - Поддерживает логические операции `&&`, `||`, `!`
   - Поддерживает сравнения `==`, `!=`
   - Значение по умолчанию `false`
   - При выводе через `print` отображается как `1` (true) и `0` (false)

------------------------------------------------------------------------

## 3. Операторы

### 3.1 Арифметика

| Оператор  | Операнды           | Семантика                         |
| --------- | ------------------ | --------------------------------- |
| `-` (унар) | `int`, `float`    | Отрицание значения                |
| `+`, `-`, `*`, `/`, `%` | `int` с `int`, `float` с `float` | Арифметика |
| `+`       | `string` с `string`| Конкатенация                      |
| Разные типы | —                | Запрещено (ошибка проверки типов) |

Применение унарного минуса к `string` или `bool` — ошибка проверки типов.

### 3.2 Операторы сравнения равенства

Операторы `==` и `!=` возвращают `bool`.

| Типы операндов      | Семантика                                      |
| ------------------- | ---------------------------------------------- |
| `int` с `int`       | Сравнение по численному значению               |
| `float` с `float`   | Сравнение по значению IEEE 754                 |
| `string` с `string` | Лексикографическое сравнение                   |
| `bool` с `bool`     | Сравнение значений                             |
| Разные типы         | Запрещено (ошибка проверки типов)              |

### 3.3 Операторы сравнения отношения

Операторы `<`, `>`, `<=`, `>=` возвращают `bool`.

| Типы операндов      | Семантика                                      |
| ------------------- | ---------------------------------------------- |
| `int` с `int`       | Сравнение по численному значению               |
| `float` с `float`   | Сравнение по значению IEEE 754                 |
| `string` с `string` | Лексикографическое сравнение                   |
| `bool`              | Запрещено                                       |
| Разные типы         | Запрещено (ошибка проверки типов)              |

### 3.4 Логические операторы

Логические операторы определены только для `bool` и возвращают `bool`.

| Оператор   | Семантика                                   |
| ---------- | ------------------------------------------- |
| `!a`       | Логическое отрицание                        |
| `a && b`   | Логическое «и» (короткая схема)             |
| `a \|\| b`   | Логическое «или» (короткая схема)           |

#### Короткая схема (short-circuit evaluation)

Операторы `&&` и `||` вычисляют второй операнд только если результат не определяется первым:

  - `a && b`: если `a == false` → результат `false`, `b` не вычисляется.
  - `a || b`: если `a == true`  → результат `true`,  `b` не вычисляется.

Применение `&&`, `||`, `!` к не-`bool` типам — ошибка проверки типов.

## 4. Переменные

| Категория | Свойство     |
| --------- | ------------ |
| `var`     | Изменяемые   |
| `const`   | Неизменяемые |


Правила:

 1.  Переменная должна быть объявлена до использования.
 2.  Тип выражения при присваивании должен совпадать с типом переменной.
 3.  Повторное присваивание `const` запрещено.

------------------------------------------------------------------------

## 5. Области видимости
 1.  Каждый блок `{}` создаёт новую область видимости
 2.  Вложенные объявления скрывают внешние с тем же именем
 3.  Дублирование объявления в одной области запрещено
 4.  Тело пользовательской функции — отдельная область видимости, в которой
     не видны переменные из вызывающего кода (только параметры функции).
 5.  Пользовательские функции и встроенные функции видны во всей программе.

------------------------------------------------------------------------


## 6. Инструкции

Допустимые инструкции (`statement`): объявление переменной, объявление константы,
присваивание, `print`, `read`, ветвление `if`, цикл `while`, `break`, `continue`,
вызов пользовательской функции, `return`, вложенный блок.

Запрещено:
 - Пустая инструкция (`;` без тела)
 - Произвольное выражение как инструкция (`-5;`, `x + 1;` и т.п.)

Вызов функции в виде инструкции допускается даже если функция возвращает значение —
в этом случае возвращённое значение отбрасывается.

## 7. Присваивание
 1.  Тип выражения должен совпадать с типом переменной
 2.  Присваивание const запрещено

------------------------------------------------------------------------

## 8. Контекстно-зависимые правила
 1.  В программе должен существовать ровно один блок `main`.
 2.  Имена пользовательских функций уникальны и не пересекаются с
     именами встроенных функций (`length`, `substr`).
 3.  В каждой пользовательской функции, объявленной с непустым типом
     возврата, любое выполнение должно завершаться инструкцией `return`
     с выражением подходящего типа (статически проверяется по обходу AST).
 4.  Инструкции `break` и `continue` допустимы только внутри тела `while`.
 5.  Инструкция `return` допустима только внутри тела пользовательской
     функции.
 6.  Условие в `if` и `while` должно иметь тип `bool`.

## 9. Print / Read
  - Read
  1. Значение читается из стандартного входа как строка.
  2. Выполняется преобразование к типу переменной.
  3. Если преобразование невозможно — ошибка времени выполнения.
  4. Использование с const запрещено.
  5. Чтение в `bool` не поддерживается.

  - Print
  1. Выражение вычисляется.
  2. Результат преобразуется в строковое представление.
  3. Значение выводится в стандартный вывод.
  4. `bool` выводится как `1` (true) или `0` (false).

------------------------------------------------------------------------

## 10. Встроенные функции

Встроенные функции являются частью языка и не требуют объявления. Они используются в выражениях как `builtin_call`.

| Функция                        | Типы аргументов                       | Тип результата | Описание                                          |
| ------------------------------ | ------------------------------------- | -------------- | ------------------------------------------------- |
| `length(s)`                    | `string`                              | `int`          | Возвращает количество Unicode code points в строке |
| `substr(s, start, length)`     | `string`, `int`, `int`                | `string`       | Возвращает подстроку из `s`, начиная с позиции `start` (в code points, с нуля), длиной `length` code points |

Правила:

 1. Аргументы должны иметь типы, совпадающие с ожидаемыми — неявных преобразований нет.
 2. Передача аргумента неверного типа — ошибка проверки типов.
 3. Встроенные функции могут использоваться в любом месте, где допустимо выражение.
 4. Для `substr`: индексация с нуля; если `start < 0`, `length < 0` или `start + length > length(s)` — ошибка времени выполнения.

------------------------------------------------------------------------

## 11. Пользовательские функции

### 11.1 Объявление

```text
func имя(параметр1: тип1, параметр2: тип2): тип_возврата { тело }
```

Тип возврата может быть одним из базовых типов (`int`, `float`, `string`, `bool`)
или ключевым словом `void` (отсутствие возвращаемого значения).

Правила:

 1. Все объявления функций находятся на верхнем уровне программы, до блока `main`.
 2. Имя функции уникально среди всех функций программы.
 3. Имя функции не должно совпадать с именами встроенных функций (`length`, `substr`).
 4. Параметры функции — это новые переменные, доступные только внутри тела функции.
 5. Имена параметров уникальны в пределах одной функции.
 6. Параметры передаются по значению (изменение параметра в теле функции не влияет на вызывающую сторону).
 7. Параметры являются изменяемыми (как `var`).

### 11.2 Возврат значения

 1. Функция с типом возврата, отличным от `void`, обязана возвращать значение через `return выражение;`.
 2. Тип выражения в `return` должен совпадать с типом возврата функции.
 3. Функция типа `void` использует `return;` без выражения, либо завершается без явного `return` в конце тела.
 4. На любом пути выполнения функции с непустым типом возврата должен быть достижим `return` с подходящим выражением.

### 11.3 Вызов функции

 1. Количество аргументов в вызове должно совпадать с количеством параметров.
 2. Тип каждого аргумента должен совпадать с типом соответствующего параметра.
 3. Функцию можно вызвать как выражение (если возвращаемый тип не `void`), либо как инструкцию.
 4. Если вызов функции, возвращающей значение, сделан как инструкция, возвращённое значение отбрасывается.
 5. Вызов функции типа `void` как выражения — ошибка проверки типов.

### 11.4 Видимость

 1. Все пользовательские функции видны во всём `main` и во всех функциях, вне зависимости от порядка объявления (взаимная и прямая рекурсия допускаются).
 2. Имена параметров и локальных переменных не выходят за пределы тела функции.

------------------------------------------------------------------------

## 12. Ветвления и циклы

### 12.1 if / else

```text
if (условие) { ... }
if (условие) { ... } else { ... }
if (условие) { ... } else if (другое_условие) { ... } else { ... }
```

 1. Тип выражения-условия должен быть `bool`. Иной тип — ошибка проверки типов.
 2. Если условие истинно, выполняется ветвь `then`, иначе — ветвь `else` (если есть).
 3. Каждая ветвь — это блок `{ ... }`, образующий собственную область видимости.

### 12.2 while

```text
while (условие) { ... }
```

 1. Тип выражения-условия должен быть `bool`. Иной тип — ошибка проверки типов.
 2. Условие проверяется перед каждой итерацией.
 3. Тело цикла — это блок `{ ... }`, образующий собственную область видимости.

### 12.3 break / continue

 1. `break;` завершает ближайший охватывающий цикл `while` и продолжает выполнение
    после цикла.
 2. `continue;` пропускает оставшуюся часть тела ближайшего охватывающего цикла
    и переходит к проверке условия следующей итерации.
 3. Использование `break` или `continue` вне тела `while` — ошибка проверки.
